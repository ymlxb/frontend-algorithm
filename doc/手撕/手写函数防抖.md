
防抖的使用条件：
  防抖会降低用户体验，需满足三个条件才使用：一是短时间内高频调用同一函数（高频事件）；二是函数为耗时操作（CPU 密集型如布局计算、IO 密集型如网络传输）；三是以最后一次调用结果为准。

防抖原理：电梯关门模型
  类比电梯关门：电梯开门后等待2秒关门若期间有人进入则重新计时，最终仅执行一次关门操作。防抖通过设置等待时间，每次触发时清除之前的定时器并重新计时，停止接触后执行最后一次调用。

防抖函数实现与优化：
  基础实现：定义全局定时器变量，每次触发时清除定时器，设置新定时器延迟执行目标函数。通过防抖函数需传入目标函数和等待时间，放回防抖处理后的函数。优化点包括透传 this 指向(使用箭头函数绑定)和参数(通过剩余参数接收并传递)。

  ```js
      function debounce(fn, delay) {
        
        let timerId;
        return () => {
          clearTimeout(timerId);
          timerId = setTimeout(() => {
            fn();
          },delay)
        }
      }

      
let count = 0;
const log = () => {
  count++;
  console.log(`函数执行第 ${count} 次，时间:`, new Date().toLocaleTimeString());
};

// 创建防抖函数
const debouncedLog = debounce(log, 1000);

// 快速连续调用
console.log('快速连续调用5次:');
debouncedLog();
debouncedLog();
debouncedLog();
debouncedLog();
debouncedLog();

// 等待结果
setTimeout(() => {
  console.log('1.5秒后 count =', count); // 应该是 1
}, 1500);
  ```
